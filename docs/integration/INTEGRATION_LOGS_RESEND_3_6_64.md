# 外部連携ログ閲覧・再送機能 v3.6.64

## 目的

外部API連携で送信・受信した履歴を管理画面から確認できるようにします。
送信失敗した outbound ログは、保存済みリクエストJSONを使って再送できます。

## 追加画面

- `/admin/integration_logs.php`
- 管理メニュー: `外部連携ログ`

## 検索できる項目

- 方向: 送信 / 受信
- 結果: 成功 / 失敗
- 連携先サイトキー
- イベント種別
- HTTPステータス
- 代理店コード、代理店名、共通ID、エラー内容、送信/応答JSONのキーワード検索

## 再送仕様

再送できるのは以下を満たすログだけです。

- `direction = outbound`
- `success = 0`
- `endpoint` が保存されている
- `request_body` が保存されている

再送時は、ログに保存された `request_body` をJSONとして読み取り、現在の外部API連携設定に登録されている同じ `site_key` の送信用APIキーでPOSTします。
そのため、APIキーを変更した後でも、現在の有効なキーで再送できます。

再送結果は、新しい `integration_event_logs` として追加保存されます。
再送元ログIDは、送信JSON内の `_retry.source_log_id` に入ります。

## 注意

連携先サイトが削除済み、または送信用APIキーが未設定の場合は再送できません。
その場合は、管理画面の「外部API連携」で連携先を再設定してから再送してください。
