# 外部API連携先の追加・運用メモ

Version: 3.6.55

## 目的

この代理店システムを中心に、`sengoku-rr.com` 以外の外部サイトにも代理店情報を送信できるようにします。

管理画面で外部サイトを複数登録すると、代理店の承認・登録・更新・停止・削除などのイベントが、有効な連携先すべてに送信されます。

## 管理画面

管理者メニュー:

```text
管理画面 > 外部API連携
```

## APIキーは連携先ごとに2種類あります

| キー | 発行元 | 登録・共有先 | 用途 |
|---|---|---|---|
| AI側が発行する受信用APIキー | 代理店システム | 対象の外部サイト側に共有 | 外部サイトから代理店システムへ登録・更新する |
| 連携先サイトが発行した送信用APIキー | 外部サイト | 代理店システムの連携先設定に登録 | 代理店システムから外部サイトへ登録・更新する |

`AI側が発行する受信用APIキー` は、連携先サイトを追加すると自動発行されます。
外部API連携画面の「連携先サイト一覧」から、接続先ごとに表示・コピー・再発行できます。

旧バージョン互換用の共通キーも残っていますが、新しい連携では使わず、接続先ごとのAI発行キーを使ってください。

## 追加する項目

| 項目 | 説明 |
|---|---|
| サイトキー | 外部サイトを識別するキーです。例: `sengoku-rr`, `school-portal` |
| 連携先名 | 管理画面に表示する名前です。例: `sengoku-rr.com` |
| 送信先URL | 外部サイトの受信URLです。ドメインだけでも入力できます。 |
| AI側が発行する受信用APIキー | 外部サイトからAIへ送る時のキーです。追加時に自動発行されます。 |
| 連携先サイトが発行した送信用APIキー | AIから外部サイトへ送る時のキーです。外部サイト側で発行してください。 |
| 状態 | `有効` の連携先にだけ送信します。 |
| 表示順 | 管理画面での並び順です。 |

## 送信先URLの入力ルール

通常はドメインだけで問題ありません。

```text
https://example.com
```

この場合、システム側で自動的に以下へ送信します。

```text
https://example.com/api/integrations/agencies
```

外部サイト側が独自のエンドポイントを使う場合は、完全なURLを入力してください。

```text
https://example.com/api/integrations/agencies
```

## 外部サイト側に必要な実装

外部サイト側は、以下のPOST APIを用意してください。

```http
POST /api/integrations/agencies
x-api-key: {連携先サイトが発行した送信用APIキー}
Content-Type: application/json
```

本システムは互換性のため、以下のヘッダーも同時に送ります。

```http
Authorization: Bearer {連携先サイトが発行した送信用APIキー}
```

## 接続テスト

管理画面の「接続テスト」ボタンを押すと、以下のようなテストデータを送信します。

```json
{
  "event": "connection_test",
  "dry_run": true,
  "source": "sengoku-ai",
  "external_id": "__connection_test__"
}
```

外部サイト側は、このデータを本登録せず、認証と受信可否だけを確認して `200` 系レスポンスを返してください。

## 既存設定との関係

旧画面の単一設定:

```text
外部代理店システムURL
外部代理店システムAPIキー
```

は互換用に残しています。

3.6.51 のマイグレーションを適用すると、旧設定に登録済みの `sengoku-rr.com` 情報は、新しい「外部API連携」画面の連携先として自動移行されます。

今後、新しい商品・外部サイトを追加する場合は「外部API連携」画面で連携先を追加してください。
